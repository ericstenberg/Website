Setting Up This Website

HTML and CSS form the backbone of this website. JavaScript and Python will be added at a later date to increase the interactivity of the site. For a fuller understanding of the interaction of the elements underlying the site, right-click on any of these pages and click "View Page Source" or similar in whatever browser you use.

The basic elements used on this website are below. By no means is this an exhaustive list, but should be used as a starting point. Let's begin!

HTML (Hypertext Markup Language)

<!DOCTYPE html>

This element instructs the browser on the version of HTML to use. This webpage is written in HTML5, as designated by "!DOCTYPE html."1

<html>...</html>

This element contains all of the other elements that make up the webpage and designates the document as a webpage.2

<head>...</head>

The first of the two main elements on a webpage, the head contains all of the title, style, and meta elements that set the flavor of the webpage.3 This element is not directly visible to the user, but does have a direct influence on how the user experiences the site.

<title>...</title>

The title is the name of the webpage that appears in the top bar and tabs of a browser.4 This is a good indicator to let the user know exactly what part of the site they are visiting.

<meta charset="UTF-8">...</meta>

The meta element assigns metadata to the webpage and can be used to give the webpage specific characteristics. For instance, this webpage has been given a charset of "UTF-8," which instructs the browser to expand the characters that will show up beyond the standard ANSI character set.5

<meta name="viewport" content="width=device-width,initial-scale=1.0">...</meta>

Another meta setting, this one sets the width of the page to the width of the browser screen.6 This makes it viable for use on desktops, laptops, phones, and other devices.

<link rel="stylesheet" href="styles.css">

One of the most important elements, the link element in this context is used to import the CSS (Cascading Style Sheets) file into the HTML file.7 Though styling could be called out on each page, this would be inefficient and would represent a problem every time a style change needed to be made across multiple pages. For larger websites with potentially hundreds of pages, this goes from cumbersome to impossible.

<body>...</body>

The second of the two main elements on a webpage, the body contains all of the information that is directly accessible by the user.8 This can be as simple as characters on the page up to different spacings, columns, headings, links, and so forth.

This webpage is split into 3 sections: header, main, and footer. Note that head, header, and heading are different elements. The head has already been covered and the header and headings will be covered later. Main and footer will also be covered further down.

<header>...</header>

Like a standard document, this website is split into 3 constitutant parts: header, main, footer. The header contains elements at the top of the page that do not change.9 At the moment, this only includes the visible title. However, a search bar will be added at a later date.

<main>...</main>

The main part of the body is where the meat of the website is contained. This is where the main posts, previews, and summaries of the pages are located.10

<footer>...</footer>

Much like the header, the footer does not or rarely ever changes. Unlike the header, it is generally located near the bottom of the page.11 This contains the links around the basics of the website (home, about me, contacts, disclaimer, etc.). If you are ever lost, just click the link home.

<div id="...">...</div>

The "div" element is a catch-all element that allows the user to create various styles depending on the website needs.12 For this website's purposes, the div id="Container" is not immediately visible, but its subelements div id="ColumnCenterHome", div id="ColumnCenterOther", div id="ColumnLeft", and div id="ColumnRight" separate different information into sections. This allows the site to have easy access to other parts of the site and makes reading the core information easier.

<a href="">...</a>

This is the basic hyperlink element. A core component of almost all websites, this allows the user to navigate different pages and download items.13

<h1>, <h2>, <hn>,...</hn>, </h2>, </h1>

Another basic element of almost all websites, headings allow the user to differentiate between titles, subtitles, etc. From 1 to 6, they decrease in size and can be modified like other elements in stylesheets, which are covered below.14

<ul>...</ul>

This element is the unordered list element, which can be found in the Tags below. This can be used to create a list where the order of elements is not important.15 See <li>...</li> below for information on adding individual elements to the unordered list.

<ol>...</ol>

This element is the ordered list element. Unlike the unordered list, this can be used to create a list where the order of elements is important.16 See <li>...</li> below for information on adding individual elements to the ordered list.

<li>...</li>

In order to create a list, elements must be added to the ordered or unordered block and this element does that. Each <li>...</li> pair corresponds to one item on the list.17

<sup>...</sup>

This element superscripts whatever text is located inside the element.18 This is useful for adding references to text, as is done on this page.

CSS (Cascading Style Sheets)

SASS

Styling with CSS is a good method for separating the out the stylistic properties of a website from the information and all-around good way of organizing a website. One of the weaknesses of CSS is the inability to set a variable once and then use that variable across a number of different elements. Enter SASS. SASS allows for the user to create CSS files with variables and flow control.

On Linux, I compile my SASS files into CSS files using the following command:

sass sassFile.scss cssFile.css

$variable: value

The $ symbol combined with a variable and value allows the developer to assign that value across a number of different properties. For example, $ColumnHeight: 100% specifies that all elements that input the ColumnHeight variable will span 100 % of their element that contains them.

padding

This property creates an empty layer between elements. This allows for better organization and less clutter on a webpage, increasing readability.

height

This property sets the height of an element in relation to its parent element.

width

This property set the width of an element in relation to its parent element.

position

This property determines how elements are positioned. For example, the "relative" position is used on this page, which allows the developer to change its position relative to where it defaults.

This is often used in conjunction with properties such as "top" or "bottom" and a numerical value to determine the magnitude of the modification.

bottom

This property is always used in conjunction with a position property. It affects the position of the element in relation to the bottom of its parent element.

text-align

Just as it says, this property aligns the text within its element.

border

Just as it says, this property creates a border around the element.

margin

Adding margins elements creates space around those elements.

font-size

Just as it states, font-size controls the size of the text.

color

Just as it states, the color property controls the color of the element. For this website, it is used to turn hyperlinks black.

text-decoration

Text-decoration adds or removes stylist elements from text. For this website, it is used to remove the underline from hyperlinks.

display

Lastly, the display property controls how elements are displayed. In the context of this website, display is used on the center columns to allow them to fit between the left and right columns on the home page and to the right on all other pages.
